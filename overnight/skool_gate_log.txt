## Skool admin API audit

FINDING: There is no public Skool admin API. Only credentials we have are
Andy's own SKOOL_EMAIL/SKOOL_PASSWORD + SKOOL_COMMUNITY_SLUG. All current
Skool tooling (skool/skool-extension/) works by scraping the live web UI
via a Chrome extension that reads conversation text from open DMs.

IMPLICATION: The goal's 'Skool admin API' clause has to be replaced with
a realistic data source. Two viable paths, building both:

  A) Manual allowlist (Mike runs the CLI when onboarding). Reliable, no
     scraping, but human-in-the-loop.
  B) Chrome-extension hook: extension scrapes Skool member list when Andy
     has it open, POSTs to /api/admin/sync-members. Automation if Andy
     keeps the tab open.

  For DM-code verify specifically, the same problem applies: we can't
  read DMs via API. Built as: (a) Chrome ext detects DM containing code
  and POSTs verification, (b) Andy/Mike runs CLI to approve manually.
  From the user's POV the contract is identical.
